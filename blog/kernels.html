Macro vs Micro Kernels and an intro to hybrid kernels

A comparative report on the different types of kernels.

The kernel is the core part of an operating system, acting as the interface between hardware and the programs running on it. It manages resources including CPU, memory, disk I/O, and networking.

Microkernels

A microkernel keeps only the minimum required functions in the kernel, such as memory handling, scheduling, and inter-process communication. Its smaller size and isolation can improve security and extensibility, but IPC overhead can reduce efficiency.

Macrokernels

A macrokernel, also known as a monolithic kernel, runs kernel services and system calls in the same address space. This can be efficient, but the larger privileged codebase can make failures more damaging.

Hybrid kernels

Hybrid kernels combine both ideas by keeping some components in kernel space and some outside it. The goal is to balance performance, modularity, and stability.
